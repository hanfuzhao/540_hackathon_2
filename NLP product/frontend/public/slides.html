Evaluation Plan

① Sarcastic Sentiment Flip Test

10 "positive-worded" insults — phrases that sound nice but mean the opposite.
Standard classifiers label these as Sincere. Ours shouldn't.

Pass threshold≥ 70%

Example: "Oh wow, you're absolutely brilliant for doing that."

② Contextual Consistency Audit

A single sarcastic remark might be harmless.
Repeated patterns across 5 messages reveal systemic abuse.

  • Counts toxic messages in window
  • Flags escalation when ≥2 of last 3 are toxic
  • Dominant pattern detection
Gaslighting escalation Persistent PA

③ Held-out Test Set (F1 / Precision / Recall)

Per-class classification report on curated labeled examples.
Confusion matrix reveals which classes the model conflates.

④ Stress Tests (slang, emoji, fragments)

Custom edge-case inputs: incomplete sentences, mixed-language, emoji substitutions.
Surfaces robustness failures before deployment.

What Evaluation Revealed

Failure: Zero-shot misses subtle sarcasm

bart-large-mnli scored "Oh wow, just wow" as Sincere (68% confidence).
Flip-test accuracy: 54% — well below 70% threshold.

Fix: Hybrid rule boost

Adding 30% regex pattern weight for zero-shot mode pushed accuracy to 78%.
Rules anchor the model on known gaslighting phrases.

Fix: Fine-tuning

Fine-tuned RoBERTa on curated data.
Reduced model weight in blend to 15% rules (model is more reliable).
Further lifts F1 on PA class.

Failure: Single-message blind spots

"Whatever, fine." alone = low-confidence PA.
3rd time in 5 messages → escalation flag triggers.

Insight: Confusion matrix

Gaslighting ↔ Passive-Aggressive most confused pair.
Guided us to add more discriminating pattern phrases to the rule library.

Insight: Context window

Evaluation showed that individual message accuracy ≠ user safety.
Conversation-level audit catches what single-turn analysis misses.
